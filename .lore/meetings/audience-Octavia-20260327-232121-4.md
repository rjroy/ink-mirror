**CRAFT NUDGE SPEC REVIEW**
Octavia (Chronicler)
2026-03-28

---

Reviewed the craft-nudge specification, which defines an on-demand daemon endpoint for surfacing where a writer's text collides with established craft principles. The nudge returns 3-5 Socratic questions grounded in specific passages, each traced to a named craft principle from writing authorities. The spec covers twelve principles organized into three detection tiers based on how much the metrics pipeline can contribute.

The discussion centered on the user path. The spec initially suggested a separate entry field for nudge submission, counter to the existing entry system and introducing friction (users would copy-paste their own writing back into the tool that holds it). Clarified the design intent: the primary path is "I'm viewing an entry I wrote, I hit a review button, and the nudge works on that entry." The daemon contract stays text-based internally (the nudge function receives text, doesn't care where it came from), but the route layer handles entry resolution so clients don't reimplement the lookup.

Updated the spec in four places. REQ-CN-3 now reflects entry resolution at the route level. REQ-CN-24 (request schema) reordered to show `entryId` as the primary path, `text` as secondary, with both optional but at least one required. REQ-CN-26 added entry reader to the route factory dependencies. Overview reframed from "writer pastes or selects text" to "writer views an entry and asks for review, or provides text directly." The nudge function itself unchanged; the entry lookup moves into the daemon's responsibility.

No open items. Spec marked approved and ready for planning.
