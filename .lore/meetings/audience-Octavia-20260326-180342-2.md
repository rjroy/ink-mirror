MEETING NOTES: Trimming architecture doc to portable pattern
Octavia | ink-mirror | 2026-03-27

SUMMARY

Reviewed `.lore/reference/architecture-pattern.md` to clarify what is truly portable infrastructure versus Guild Hall-specific implementation detail. The document had accumulated prescriptive guidance on workers, git isolation, commissions/meetings, and five-layer service decomposition that shouldn't constrain future projects adopting the pattern.

Three critical reframes emerged: (1) the daemon is the application itself, not a boundary between clients and the real system—web and CLI are dumb views into it; (2) the SDK Runner is a principle (one centralized entry point for LLM calls) not a specific function signature to copy; (3) the CLI matters because it makes the daemon usable by agents, not just humans. "When you make a thing, you make a CLI" is a design philosophy, not a convenience feature.

Stripped the document from 299 to ~120 lines by removing workers, activation contracts, three-branch git model, activity type state machines, five-layer architecture, memory system, and production initialization order. Kept the portable parts: daemon as app, DI-based route factories, operations registry with runtime discovery, EventBus/SSE streaming, file-based state, type boundaries, and testing seams via dependency injection.

DECISIONS MADE

Cut five-layer service architecture (Records → Lifecycle → Workspace → Session Prep → Orchestrator) as too specific to Guild Hall's complexity. New projects should derive their own layering from their domain.

Reframed document title and opening statement to emphasize daemon-as-app philosophy over boundary/client terminology.

SDK Runner remains in the doc but as a design principle (centralized session entry point) rather than specific function signatures or the current event translator/toolbox tier system.

ARTIFACTS

`.lore/reference/architecture-pattern.md` — revised to remove Guild Hall-specific infrastructure and refocus on portable daemon-first pattern.

OPEN ITEMS

None. Session closed.
